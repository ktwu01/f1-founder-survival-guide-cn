# 文档规范（中文）

本页用于统一本仓库的写作方式，保证内容可读、可查、可验证。

## 目标

- 让读者在 1-2 分钟内判断“这条信息是否适用于我”。
- 每条关键结论都可追溯到原始来源。
- 明确边界：信息整理，不替代专业法律/税务/移民建议。

## 标准条目模板（建议复制使用）

```markdown
## 条目标题（一句话说明场景）

### 背景与问题
- 这条信息要解决什么问题？

### 适用人群
- 谁适用；谁不适用（边界条件）

### 核心结论（可执行）
- 结论 1
- 结论 2

### 风险与例外
- 常见误区
- 可能失效的情况

### 可执行下一步（Checklist）
- [ ] 步骤 1
- [ ] 步骤 2
- [ ] 步骤 3

### 来源与证据
- 官方来源：
- 机构来源：
- 社区/经验来源：

### 更新时间
- YYYY-MM-DD
```

## 来源可信度分级（A/B/C）

- A 级（优先）：官方与法律文本  
  例如政府官网、官方政策页面、学校国际学生办公室官方文档。
- B 级（次优）：可信机构与专业组织  
  例如律所公开文章、知名研究机构、大学发布的解释性内容。
- C 级（辅助）：个人经验与社区讨论  
  例如论坛帖子、个人博客、社交媒体线程。

使用规则：
- 关键结论尽量由 A 级来源支撑。
- 只有 C 级时，必须显式标注“仅供参考，需二次核验”。
- 同一结论建议至少给出 2 个不同来源。

## 链接与格式规则

- 链接必须可点击，尽量使用原始链接而不是二次转载。
- 外部链接名称要清晰（不要只写“点这里”）。
- 长清单尽量表格化或 checklist 化，方便扫描和维护。
